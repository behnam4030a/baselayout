## گام ۱ — توضیح صفحه

قبل از هر چیز از کاربر بخواه توضیح دهد:
- این صفحه چیست و چه کاری می‌کند؟
- در کجای پروژه قرار می‌گیرد (صفحه جدید؟ بخشی از صفحه موجود؟ مودال؟)
- آیا داده‌ای دارد که باید داینامیک باشد؟

بدون دریافت این توضیح ادامه نده.

## گام ۲ — دریافت لینک Figma (دسکتاپ)

از کاربر بخواه لینک Figma طرح **دسکتاپ** را بدهد.

بدون دریافت لینک، هیچ کدی ننویس.

وقتی لینک را دریافت کردی:
1. با `get_design_context` طرح را بخوان
2. با `get_screenshot` تصویر واقعی طرح را ببین
3. کامپوننت‌های موجود در `components/` را بررسی کن — لیست آن‌ها:
   - `modal` — مودال slide-in از راست / centered / با پنل جانبی
   - `button` — دکمه‌های primary/secondary/outline/subtle/transparent/ai
   - `input` — فیلد ورودی با آیکن و divider
   - `select` — منوی انتخاب
   - `textarea` — متن چندخطی
   - `checkbox` — چک‌باکس
   - `radio` — دکمه رادیو
   - `switch` — تاگل سوییچ
   - `tab` — تب‌ها
   - `tag` — تگ/برچسب وضعیت
   - `table` — جدول با جستجو، فیلتر، صفحه‌بندی، عملیات دسته‌ای
   - `dialog` — دیالوگ تأیید/هشدار
   - `toast` — پیام موقت
   - `tooltip` — راهنمای tooltip
   - `message-bar` — نوار پیام
   - `label` — برچسب فرم
   - `typography` — تایپوگرافی

**قانون مهم:** هر المانی که معادل کامپوننت موجود دارد را **از همان کامپوننت** بساز. چیزی را که کامپوننتش وجود دارد از صفر نساز.

## گام ۳ — پیاده‌سازی طرح دسکتاپ

### اصول پایه پروژه:
- زبان: فارسی/RTL — `direction: rtl`
- فونت: Vazirmatn
- Design tokens از `css/style.css :root` — از متغیرهای CSS پروژه استفاده کن، رنگ hard-code نزن
- **یک طرح واحد** — طرح دسکتاپ و موبایل جدا نیست؛ همان HTML با CSS رسپانسیو داینامیک می‌شود

### قوانین RTL:
- اول DOM = سمت راست فیزیکی
- `justify-content: flex-start` = چینش از راست
- `justify-content: space-between` = اول‌ها راست، آخری‌ها چپ

### اجرا:
1. ساختار HTML را با کامپوننت‌های موجود بساز
2. استایل‌های اختصاصی را در فایل CSS مرتبط بنویس (نه inline style)
3. اگر مودال است از `components/modal/modal.js` استفاده کن و IIFE pattern در `app.js` بنویس
4. اگر جدول است از `components/table/` استفاده کن
5. JS را به صورت IIFE در `js/app.js` یا فایل مرتبط بنویس

## گام ۴ — دریافت لینک Figma رسپانسیو (موبایل)

بعد از پیاده‌سازی دسکتاپ، از کاربر بخواه لینک Figma **نسخه موبایل/رسپانسیو** را بدهد.

بدون دریافت این لینک، استایل موبایل حدس نزن.

وقتی لینک را دریافت کردی:
1. با `get_design_context` طرح موبایل را بخوان
2. با `get_screenshot` تصویر موبایل را ببین
3. تفاوت‌های layout را شناسایی کن

### پیاده‌سازی رسپانسیو:
- breakpoint اصلی: `768px` (موبایل زیر این)
- breakpoint ثانوی: `1023px` (تبلت)
- از `@media (max-width: 768px)` برای تغییرات موبایل استفاده کن
- **همان HTML** — فقط CSS تغییر می‌کند، HTML جدید نمی‌سازی
- اگر bottom nav وجود دارد: `bottom: calc(var(--bottom-nav-h) + 8px)` برای مودال‌ها

## یادآوری‌های کلی

- هرگز بدون لینک Figma کدنویسی را شروع نکن
- هرگز کامپوننت موجود را دوباره نساز — از README هر کامپوننت استفاده کن
- CSS variables پروژه: `--brand`, `--text-1`, `--text-2`, `--stroke`, `--card-bg`, `--page-bg`, `--radius-card`, `--transition`
- فایل CSS جدید نساز مگر برای صفحه/feature کاملاً جدید — به فایل CSS موجود آن بخش اضافه کن
- اگر صفحه جدید است، load order را در HTML رعایت کن:
  1. `css/style.css`
  2. کامپوننت‌های CSS
  3. `components/modal/modal.css`
  4. اسکریپت‌ها آخر body: `modal.js` → `app.js`
